# Glossary of Terms

## A

**API (Application Programming Interface)**
A set of protocols and tools for building software applications. In Frappe, APIs are automatically generated for each DocType.

**App**
A collection of DocTypes, reports, and other functionality that extends the Frappe framework. ERPNext is itself an app built on Frappe.

**Asset**
A physical or intangible item owned by a company that has monetary value and is tracked in the asset management system.

**Autoname**
A pattern or rule that automatically generates names for documents when they are created.

## B

**Bench**
The command-line tool and directory structure used to manage Frappe applications and sites. A bench contains multiple apps and sites.

**BOM (Bill of Materials)**
A comprehensive list of materials, parts, and components required to build a product.

**Bootstrap**
A front-end framework for developing responsive and mobile-first websites. Frappe uses Bootstrap for UI components.

**Background Job**
A task that runs asynchronously in the background, typically for long-running processes like sending emails or generating reports.

## C

**Child Table**
A table of related records that is embedded within a parent document. Used for one-to-many relationships.

**Client Script**
JavaScript code that runs in the browser to enhance user interface functionality.

**Controller**
A Python class that defines the behavior and business logic for a DocType.

**Custom Field**
A field added to an existing DocType without modifying the core definition.

**Custom Script**
JavaScript or Python code that extends the functionality of standard DocTypes.

## D

**DocType**
The fundamental building block in Frappe that defines the structure, behavior, and user interface for a type of document.

**Document**
An instance of a DocType, representing a single record in the database.

**Database**
The system that stores all application data. Frappe uses MariaDB/MySQL by default.

**Dashboard**
A visual interface that displays charts, reports, and key performance indicators.

**Developer Mode**
A setting that enables additional debugging features and disables caching for development purposes.

## E

**ERP (Enterprise Resource Planning)**
Software that helps organizations manage business processes and operations.

**ERPNext**
An open-source ERP system built on the Frappe framework.

**Email Template**
A predefined template for sending emails, often used for notifications and reports.

**Environment**
The set of tools, dependencies, and configuration needed to develop and run Frappe applications.

## F

**Frappe**
The open-source framework that powers ERPNext and other business applications.

**Fixture**
Data that is exported and imported to configure the system, such as custom fields, permissions, and settings.

**Form**
The user interface for creating and editing documents.

**Field**
A single piece of data within a DocType, such as text, numbers, dates, or links to other documents.

**Filter**
A condition used to limit the records displayed in reports or lists.

## G

**Git**
A version control system used to track changes in code and collaborate with other developers.

**GitHub**
A web-based hosting service for Git repositories.

**Global Search**
A feature that allows searching across multiple DocTypes simultaneously.

**Group**
A way to organize users or documents based on common characteristics.

## H

**Hook**
A mechanism that allows custom code to be executed at specific points in the application lifecycle.

**HTML (HyperText Markup Language)**
The standard markup language for creating web pages.

**HTTP**
The protocol used for communication between web browsers and servers.

**HTTPS**
The secure version of HTTP that encrypts communication.

## I

**Instance**
A running copy of the Frappe application with its own database and configuration.

**Integration**
The process of connecting Frappe with external systems and services.

**Inventory**
The stock of goods and materials held by a company.

**Invoice**
A document that requests payment for goods or services provided.

## J

**Jinja**
A template engine used by Frappe for generating HTML, email templates, and print formats.

**JavaScript**
A programming language used for client-side web development.

**Job Queue**
A system for managing background tasks and asynchronous operations.

**JSON (JavaScript Object Notation)**
A lightweight data interchange format used for APIs and configuration.

## K

**Key Performance Indicator (KPI)**
A measurable value that demonstrates how effectively a company is achieving key business objectives.

**Knowledge Base**
A repository of information about the system, including documentation and best practices.

## L

**Link Field**
A field that creates a relationship between two DocTypes.

**Log**
A record of events that occur within the system, used for debugging and auditing.

**License**
The legal terms that govern the use and distribution of software.

**Localization**
The process of adapting software for different languages and regions.

## M

**Master DocType**
A DocType that represents core data entities like customers, suppliers, or items.

**MariaDB**
A popular open-source database system used by Frappe.

**Migration**
The process of updating the database schema when installing or updating applications.

**Module**
A logical grouping of related DocTypes and functionality within an app.

## N

**Naming Series**
A pattern used to automatically generate unique names for documents.

**Node.js**
A JavaScript runtime environment used for front-end development in Frappe.

**Notification**
A message sent to users to inform them about important events.

## O

**ORM (Object-Relational Mapping)**
A technique that converts data between incompatible systems using object-oriented programming languages.

**Override**
The process of replacing standard functionality with custom code.

## P

**Patch**
A script that updates the database schema or data when applications are upgraded.

**Permission**
Rules that control what users can see and do within the system.

**Print Format**
A template that defines how documents appear when printed or exported to PDF.

**Property Setter**
A mechanism to customize field properties without modifying the core DocType.

**Python**
The programming language used for server-side development in Frappe.

## Q

**Query**
A request for data from the database.

**Queue**
A system for managing background tasks and asynchronous operations.

## R

**Redis**
An in-memory data structure store used for caching and session management.

**Report**
A formatted presentation of data from the database.

**Repository**
A storage location for software code and documentation.

**Role**
A collection of permissions that can be assigned to users.

**Router**
The component that maps URLs to the appropriate code handlers.

## S

**Scheduler**
A system that runs tasks at specified intervals, such as daily or weekly.

**Script**
Code that extends the functionality of the system.

**Server**
The computer system that hosts the Frappe application and database.

**Session**
A period of interaction between a user and the application.

**Site**
A Frappe installation with its own database and configuration.

**Single DocType**
A DocType that has only one record, typically used for system settings.

**SocketIO**
A library that enables real-time communication between the server and client.

**SQL (Structured Query Language)**
The language used to communicate with relational databases.

## T

**Template**
A predefined format for generating documents, emails, or web pages.

**Transaction DocType**
A DocType that represents business transactions and goes through an approval workflow.

**Trigger**
An event that causes specific code to be executed.

**Type**
A classification that defines the characteristics and behavior of data.

## U

**UI (User Interface)**
The visual elements that users interact with in an application.

**Update**
The process of installing new versions of applications or system components.

**User**
A person who has access to the Frappe system.

**URL (Uniform Resource Locator)**
The address of a web page or resource.

## V

**Validation**
Rules that ensure data entered into the system is correct and consistent.

**Version Control**
A system that tracks changes to code over time.

**Virtual Machine**
A software-based computer that runs on a physical host.

## W

**Webhook**
A mechanism for sending real-time notifications to external systems when events occur.

**Workflow**
A sequence of steps that a document goes through during its lifecycle.

**Worker**
A process that handles background tasks and job queues.

## X

**XML (eXtensible Markup Language)**
A markup language that defines rules for encoding documents.

## Y

**YAML**
A human-readable data serialization format often used for configuration files.

## Z

**Zero Downtime**
The ability to update or maintain a system without interrupting service.
